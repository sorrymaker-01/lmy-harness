**本技能使用模板进行LBO模型构建。始终首先检查是否有附带的模板文件。**

开始任何LBO模型之前：
1. **如果附带/提供了模板文件**：完全使用该模板的结构 - 复制它并用用户数据填充

**重要**：当附带`LBO_Model.xlsx`等文件时，您必须将其用作模板 - 不要从头构建。即使模板看起来复杂或功能超出需求，也要复制它并根据用户要求进行调整。当提供模板时，绝不要决定"从头构建"。

## CLAUDE的关键说明 - 首先阅读

### 核心原则
* **每个计算必须是Excel公式** - 永远不要在Python中计算值并将结果硬编码到单元格中。模型必须是动态的，在输入变化时更新。
* **使用模板结构** - 遵循`examples/LBO_Model.xlsx`或用户提供的模板中的组织方式。不要发明自己的布局。
* **使用正确的单元格引用** - 所有公式应引用适当的单元格。永远不要输入应来自其他单元格的数字。
* **保持符号约定一致性** - 遵循模板使用的任何符号约定（有些使用负数表示流出，有些使用正数）。始终保持一致。
* **逐部分工作** - 在进入下一部分之前完全完成一个部分，因为后面的部分通常依赖于前面的部分。

### 公式颜色约定
* **蓝色（0000FF）**：硬编码输入 - 不引用其他单元格的输入数字
* **黑色（000000）**：带计算的公式 - 任何使用运算符或函数的公式（`=B4*B5`、`=SUM()`、`=-MAX(0,B4)`）
* **紫色（800080）**：链接到**同一标签页**的单元格 - 无计算的直接引用（`=B9`、`=B45`）
* **绿色（008000）**：链接到**不同标签页**的单元格 - 跨工作表引用（`=Assumptions!B5`、`='Operating Model'!C10`）

### 数字格式标准
* **货币**：`$#,##0;($#,##0);"-"`或`$#,##0.0`，取决于模板
* **百分比**：`0.0%`（一位小数）
* **倍数**：`0.0"x"`（一位小数）
* **MOIC/详细比率**：`0.00"x"`（两位小数以获得精度）
* **所有数字单元格**：右对齐

## 模板分析阶段 - 首先执行此操作

在填写任何公式之前，彻底检查模板：

1. **映射结构** - 识别每个部分的位置以及它们之间的关系。注意哪些部分输入到其他部分。

2. **了解时间线** - 哪些列代表哪些期间？是否有"Closing"或"Pro Forma"列？预测期从哪里开始？

3. **识别输入与公式单元格** - 模板通常使用颜色编码、边框或阴影来指示哪些单元格需要输入与公式。尊重这些约定。

4. **仔细阅读现有标签** - 行标签准确告诉您预期的计算。不要假设 - 阅读模板的要求。

5. **检查现有公式** - 一些模板部分预填充。除非特别要求，否则不要覆盖可工作的公式。

6. **注意模板特定约定** - 符号约定、小计结构、部分组织方式、是否有不同组件的独立标签页等。

## 填写公式 - 通用方法

对于每个需要公式的单元格，遵循此层次结构：

### 步骤1：检查模板
* 单元格是否已有公式？如果是，验证其正确性并继续。
* 是否有注释或说明指示预期计算？
* 行/列标签是否使计算显而易见？
* 相邻单元格是否显示您应遵循的模式？

### 步骤2：检查用户说明
* 用户是否指定了特定的计算方法？
* 是否有影响此公式的声明假设？
* 是否提到任何特殊要求？

### 步骤3：应用标准实践
* 如果模板和用户都未指定，使用标准LBO建模约定
* 记录您做出的任何假设
* 如果确实不确定，询问用户

## 常见问题领域

以下计算模式在LBO模型中经常引起问题。遇到这些时要特别注意：

### 平衡部分
* 当两个部分必须相等（如来源 = 用途）时，一个项目通常是"插值项"（平衡数字）
* 识别哪个项目是插值项，并将其计算为差额

### 税费计算
* 税费公式应仅引用相关的收入行和税率
* 不应引用不相关的部分（如债务时间表）
* 考虑亏损是否产生税盾或仅被忽略

### 利息和循环引用
* 如果利息引用受现金流影响的余额，可能产生循环
* 使用**期初余额**（而非平均或期末）来打破循环引用
* 模式：利息 → 现金流 → 偿还 → 期末余额（如果利息使用期末余额，这会循环回来）

### 债务偿还 / 现金清偿
* 当存在多个债务层级时，通常有优先顺序
* 现金清偿应尊重优先级瀑布
* 余额不能为负 - 适当使用MAX或MIN函数

### 回报计算（IRR/MOIC）
* 现金流必须有正确的符号：投资 = 负数，收益 = 正数
* 如果使用XIRR，需要对应的日期
* 如果使用IRR，现金流应在连续期间内
* MOIC = 总收益 / 总投资

### 敏感性分析表
* Excel的数据表功能可能无法与openpyxl配合使用
* 可能需要引用行/列标题的显式公式
* 每个单元格应显示不同的值 - 如果全部相同，公式未正确变化
* 使用混合引用（如$A5表示行输入，B$4表示列输入）

python /mnt/skills/public/xlsx/recalc.py model.xlsx

### 部分平衡
- [ ] 任何必须平衡的部分（来源/用途、资产/负债）完全平衡
- [ ] 插值项正确计算为平衡数字
- [ ] 各部分之间应匹配的金额一致

### 收入/运营预测
- [ ] 收入/顶层从驱动因素或增长率正确构建
- [ ] 所有成本和费用项目适当计算
- [ ] 小计和总计正确求和
- [ ] 利润率和比率合理
- [ ] 与假设的链接正确

### 资产负债表（如适用）
- [ ] 资产 = 负债 + 所有者权益（必须平衡）
- [ ] 所有项目链接到适当的附表或滚转
- [ ] 期初余额 = 上期期末余额
- [ ] 包含检查行并显示零

### 现金流（如适用）
- [ ] 以正确的收入数字开始
- [ ] 非现金项目适当加/减
- [ ] 营运资本变化有正确的符号
- [ ] 期末现金 = 期初现金 + 净现金流
- [ ] 现金余额在各报表间一致

### 支持附表
- [ ] 滚转附表平衡（期初 + 变动 = 期末）
- [ ] 附表正确链接到主报表
- [ ] 计算项目使用适当的驱动因素
- [ ] 所有期间计算一致

### 债务/融资附表（如适用）
- [ ] 期初余额与来源或上期一致
- [ ] 利息在适当的余额上计算（通常为期初）
- [ ] 偿还尊重现金可用性和优先级
- [ ] 期末余额不能为负
- [ ] 总计正确汇总各层级

### 回报/输出分析
- [ ] 退出/终值正确计算
- [ ] 包含所有相关调整
- [ ] 现金流符号正确（投资为负，收益为正）
- [ ] IRR/MOIC公式引用完整范围
- [ ] 结果对情景合理

### 敏感性分析表（如适用）
- [ ] 行和列标题包含适当的输入值
- [ ] 每个数据单元格包含公式（非硬编码）
- [ ] 每个数据单元格显示不同的值
- [ ] 值按预期方向变动
- [ ] 基准情景出现在标题与基准假设匹配的位置

### 格式
- [ ] 硬编码输入为蓝色（0000FF）
- [ ] 计算公式为黑色（000000）
- [ ] 同一标签页链接为紫色（800080）
- [ ] 跨标签页链接为绿色（008000）
- [ ] 所有数字右对齐
- [ ] 全程应用适当的数字格式
- [ ] 无单元格显示错误值（#REF!, #DIV/0!, #VALUE!, #NAME?）

### 逻辑合理性检查
- [ ] 数字数量级合理
- [ ] 趋势有意义（如预期的增长、下降、稳定）
- [ ] 无明显错误值（应为正数处为负、不可能的百分比等）
- [ ] 关键输出在该类型分析的合理范围内

## 应避免的常见错误

| 错误 | 出错原因 | 如何修复 |
|-------|-----------------|------------|
| 硬编码计算值 | 输入变化时模型不更新 | 始终使用引用源单元格的公式 |
| 复制后单元格引用错误 | 公式指向错误单元格 | 验证所有链接，使用适当的$锚定 |
| 循环引用错误 | 模型无法计算 | 对利息类计算使用期初余额，打破循环 |
| 部分不平衡 | 应匹配的总计不匹配 | 确保一个项目是插值项（计算为差额） |
| 不可能位置出现负余额 | 支付/使用超过可用量 | 适当使用MAX(0, ...)或MIN函数 |
| IRR/回报错误 | 符号错误或范围不完整 | 检查现金流符号并确保公式覆盖所有期间 |
| 敏感性分析表显示相同值 | 公式未随输入变化 | 检查单元格引用 - 需要混合引用($A5, B$4) |
| 滚转不一致 | 期初 ≠ 上期期末 | 验证期间之间的链接 |
| 符号约定不一致 | 加法变成减法或反之 | 全程遵循模板的约定 |
