# 三表财务模型模板完成

完成并填充集成财务模型模板，确保损益表、资产负债表和现金流量表之间的正确链接。

## 模型结构

### 识别模板标签组织

模板在标签命名约定和组织方面有所不同。在填充之前，查看所有标签以了解模板的结构。以下是常见的标签名称及其典型内容：

| 常见标签名称 | 需查找的内容 |
|------------------|----------------------|
| IS, P&L, Income Statement | 损益表 |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资金附表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销附表 |
| Debt, Debt Schedule | 债务附表 |
| NOL, Tax, DTA | 净营业亏损附表 |
| Assumptions, Inputs, Drivers | 驱动假设和输入 |
| Checks, Audit, Validation | 错误检查仪表板 |

**模板审查清单**
- 识别模板中存在哪些标签（并非所有模板都包含每个附表）
- 注意上面未列出的任何模板特定标签
- 了解标签依赖关系（例如，哪些附表输入到主要报表中）
- 定位每个标签上的输入单元格与公式单元格

### 理解模板结构

在填充模板之前，熟悉其现有布局，以确保数据输入到正确的位置，并且公式保持完整。

**识别行结构**
- 找到每个标签顶部的模型标题
- 识别部分标题及其视觉分隔
- 找到指示百万美元、百分比、倍数等的单位行
- 注意区分实际与估计期间的列标题
- 确认期间标签（例如，FY2024A，FY2025E）
- 识别输入单元格与公式单元格（通常通过字体颜色区分）

**识别列结构**
- 确认最左列中的行项目标签
- 验证历史年份在预测年份之前
- 注意分隔历史与预测期间的视觉边界
- 检查所有标签上的列顺序是否一致

**使用命名范围**
模板通常对关键输入和输出使用命名范围。在输入数据之前：
- 查看模板中的现有命名范围（Excel中的公式 → 名称管理器）
- 常见的命名范围包括：收入增长率、成本百分比、关键输出（净收入、EBITDA、总债务、现金）、情景选择器单元格
- 确保输入输入到这些命名范围的单元格中

### 预测期间
- 模板通常从最后一个历史年份向前预测5年
- 验证历史（A）与预测（E）列是否明确分开
- 确认列使用财年表示法（例如，FY2024A，FY2025E）

## 边际分析

**注意：以下边际分析仅在用户提示或模板明确要求时执行。如果没有提示，请跳过此部分。**

在损益表（IS）标签上计算并显示盈利能力边际，以跟踪运营效率并实现同行比较。

### 需包含的核心边际

| 边际 | 公式 | 衡量内容 |
|--------|---------|------------------|
| 毛利率 | 毛利润 / 收入 | 定价能力、生产效率 |
| EBITDA边际 | EBITDA / 收入 | 核心运营盈利能力 |
| EBIT边际 | EBIT / 收入 | 扣除折旧与摊销后的运营盈利能力 |
| 净收入边际 | 净收入 / 收入 | 底线盈利能力 |

### 带边际的损益表布局

在每个利润行项目下方直接显示边际百分比：
- 毛利润下方显示毛利率%
- EBIT下方显示EBIT边际%
- EBITDA下方显示EBITDA边际%
- 净收入下方显示净收入边际%

## 信用指标

**注意：以下信用分析仅在用户提示或模板明确要求时执行。如果没有提示，请跳过此部分。**

在资产负债表（BS）标签上计算并显示信用/杠杆指标，以评估财务健康状况、债务容量和 covenant 合规性。

### 需包含的核心信用指标

| 指标 | 公式 | 衡量内容 |
|--------|---------|------------------|
| 总债务 / EBITDA | 总债务 / 最近12个月EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | (总债务 - 现金) / 最近12个月EBITDA | 扣除现金的杠杆 |
| 利息覆盖率 | EBITDA / 利息支出 | 偿债能力 |
| 债务 / 总资本 | 总债务 / (总债务 + 权益) | 资本结构 |
| 债务 / 权益 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | (流动资产 - 存货) / 流动负债 | 即时流动性 |

### 信用指标层次检查

验证上行情景显示最强的信用状况：
- 杠杆：上行 < 基准 < 下行（越低越好）
- 覆盖率：上行 > 基准 > 下行（越高越好）
- 流动性：上行 > 基准 > 下行（越高越好）

### Covenant合规跟踪

如果已知债务covenant，添加明确的合规检查，将实际指标与covenant阈值进行比较。

## 情景分析（基准/上行/下行）

在假设标签中使用带有CHOOSE或INDEX/MATCH公式的情景切换（下拉菜单）。

| 情景 | 描述 |
|----------|-------------|
| 基准情景 | 管理层指导或共识估计 |
| 上行情景 | 超出指导的增长，边际扩张 |
| 下行情景 | 低于趋势的增长，边际压缩 |

**需敏感性分析的关键驱动因素**：收入增长、毛利率、SG&A%、DSO/DIO/DPO、资本支出%、利率、税率。

**情景审计检查**：切换所有报表，所有情景中的BS余额，现金匹配，层次保持（上行 > 基准 > 下行，适用于NI、EBITDA、FCF、边际）。

## 国内财务报告数据提取

如果模板特别要求从国内财务报告（年报、半年报、季报）中提取数据，请参阅[references/financial-reports.md](references/financial-reports.md)获取详细的提取指南。仅在使用国内上市公司数据填充模板时才需要此参考。

## 完成模型模板

本节提供完成任何三表财务模型模板的一般指导，同时保留现有公式并确保数据完整性。

### 步骤1：分析模板结构

在输入任何数据之前，彻底审查模板以了解其架构：

**识别输入与公式单元格**
- 寻找区分输入单元格与公式单元格的视觉线索（字体颜色、单元格阴影）
- 常见约定：蓝色字体 = 输入，黑色字体 = 公式，绿色字体 = 链接到其他工作表
- 使用Excel的追踪先例/依赖项（公式 → 追踪先例）来了解单元格关系
- 检查可能控制关键输入的命名范围（公式 → 名称管理器）

**映射模板流程**
- 识别哪些标签输入到其他标签（例如，假设 → IS → BS → CF）
- 注意任何支持附表及其与主要报表的链接
- 在填充之前记录模板的特定行项目和结构

### 步骤2：填充数据而不破坏公式

**数据输入黄金规则**

| 规则 | 描述 |
|------|-------------|
| 仅编辑输入单元格 | 除非有意替换公式，否则永远不要覆盖包含公式的单元格 |
| 保留单元格引用 | 复制数据时，使用粘贴值（Ctrl+Shift+V）以避免用源格式覆盖公式 |
| 匹配模板单位 | 在输入数据之前验证模板是否使用千、百万或实际值 |
| 尊重符号约定 | 遵循模板的现有符号约定（例如，费用为正数或负数） |
| 检查循环引用 | 如果模板使用迭代计算，确保启用迭代计算 |

**安全数据输入流程**
1. 识别指定用于输入的确切单元格（通常突出显示或标记）
2. 首先输入历史数据，然后验证公式在这些期间是否正确计算
3. 输入为预测计算提供输入的假设驱动因素
4. 审查计算输出以确认公式按预期工作
5. 如果必须修改公式单元格，在进行更改前记录原始公式

**处理预建公式**
- 如果公式引用尚未填充的单元格，预计在完成所有输入之前会出现临时错误（#REF!、#DIV/0!）
- 当公式产生意外结果时，追踪先例以识别缺失或不正确的输入
- 在检查所有标签上的公式依赖项之前，永远不要删除行/列

### 步骤3：验证公式

**公式完整性检查**

在依赖模板输出之前，验证公式是否正常运行：

| 检查类型 | 方法 |
|------------|--------|
| 追踪先例 | 选择公式单元格 → 公式 → 追踪先例以验证其引用正确的输入 |
| 追踪依赖项 | 验证关键输入流向预期的输出单元格 |
| 评估公式 | 使用公式 → 评估公式逐步执行复杂计算 |
| 检查硬编码 | 预测公式应引用假设，而不是包含硬编码值 |
| 用已知值测试 | 输入简单的测试值以验证公式产生预期结果 |
| 跨标签一致性 | 确保相同的公式逻辑适用于所有预测期间 |

**需注意的常见公式问题**
- 混合绝对/相对引用在跨期间复制时导致不正确的结果
- 指向外部文件或已删除范围的断开链接（#REF!错误）
- 在收入增长前的早期期间除以零（#DIV/0!错误）
- 循环引用警告（对于利息计算可能是有意的）
- 预测列之间的公式不一致（使用Ctrl+\查找差异）

**验证跨标签链接**
- 确认出现在多个标签上的值是链接的（不是重复的）
- 验证附表总计与主要报表上的相应行项目匹配
- 检查所有标签上的期间标签是否对齐

### 步骤4：按工作表进行质量检查

在填充模板后对每个工作表执行这些验证检查：

**损益表（IS）质量检查**
- 历史期间的收入数字与源数据匹配
- 所有费用行项目总和等于报告的总计
- 小计（毛利润、EBIT、EBT、净收入）计算正确
- 税收计算逻辑适当（正确处理亏损）
- 预测驱动因素引用假设标签（无硬编码）
- 期间间变化方向合理

**资产负债表（BS）质量检查**
- 每个期间的资产 = 负债 + 权益（主要检查）
- 现金余额与现金流量表期末现金匹配
- 营运资金账户与支持附表匹配（如适用）
- 留存收益正确结转：前期RE + 净收入 - 股息 +/- 调整 = 期末RE
- 债务余额与债务附表匹配（如适用）
- 所有资产负债表项目具有适当的符号（资产为正，大多数负债为正）

**现金流量表（CF）质量检查**
- CFO顶部的净收入与损益表净收入匹配
- 非现金加回（D&A、SBC等）与其源附表/报表匹配
- 营运资金变化具有正确的符号（资产增加 = 现金使用 = 负）
- 资本支出与PP&E附表或固定资产结转匹配
- 融资活动与BS上的债务和权益账户变化匹配
- 期末现金与资产负债表现金匹配
- 期初现金等于前期期末现金

**支持附表质量检查**
- 期初余额等于前期期末余额
- 结转逻辑完整（期初 + 增加 - 减少 = 期末）
- 附表总计与主要报表行项目匹配
- 计算中使用的假设与假设标签匹配

### 步骤5：跨报表完整性检查

在验证单个工作表后，确认三个报表正确集成：

| 检查 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金匹配 | CF期末现金 - BS现金 | = 0 |
| 净收入链接 | IS净收入 - CF起始净收入 | = 0 |
| 留存收益 | 前期RE + NI - 股息 - BS期末RE | = 0（根据需要调整SBC/其他项目） |

### 步骤6：最终审查

在认为模型完成之前：
- 切换所有情景（如适用）以验证每个案例中的检查都通过
- 审查所有#REF!、#DIV/0!、#VALUE!和#NAME?错误并解决或记录
- 确认所有输入单元格已填充（搜索占位符值）
- 验证所有标签上的单位一致
- 在进行任何其他修改之前保存干净版本

## 模型验证和审计

本节整合了已完成模板的所有验证检查和审计程序。

### 核心链接（必须始终保持）

有关所有公式详细信息，请参阅[references/formulas.md](references/formulas.md)。

| 检查 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金匹配 | CF期末现金 - BS现金 | = 0 |
| 现金月度与年度 | 期末现金（月度） - 期末现金（年度） | = 0 |
| 净收入链接 | IS净收入 - CF起始净收入 | = 0 |
| 留存收益 | 前期RE + NI + SBC - 股息 - BS期末RE | = 0 |
| 权益融资 | Δ普通股/APIC（BS） - 权益发行（CFF） | = 0 |
| 第0年权益 | 权益筹集（第0年） - 期初权益资本（第1年） | = 0 |

### 符号约定参考

| 报表 | 项目 | 符号约定 |
|-----------|------|-----------------|
| CFO | D&A, SBC | 正（加回） |
| CFO | ΔAR（增加） | 负（现金使用） |
| CFO | ΔAP（增加） | 正（现金来源） |
| CFI | 资本支出 | 负 |
| CFF | 债务发行 | 正 |
| CFF | 债务偿还 | 负 |
| CFF | 股息 | 负 |

### 循环引用处理

利息支出创建循环：利息 → 净收入 → 现金 → 债务余额 → 利息

在Excel中启用迭代计算：文件 → 选项 → 公式 → 启用迭代计算。将最大迭代次数设置为100，最大变化设置为0.001。在假设标签中添加断路器切换。

### 检查类别

**第1节：货币一致性**
- 货币在假设中识别和记录
- 所有标签使用一致的货币符号和比例
- 单位行与模型货币匹配

**第2节：资产负债表完整性**
- 资产 = 负债 + 权益（每个期间）
- 公式：资产 - 负债 - 权益（必须 = 0）

**第3节：现金流量完整性**
- 现金与BS匹配（CF期末现金 = BS现金）
- 现金月度与年度：期末现金（月度） = 期末现金（年度）
- NI与IS匹配（CF净收入 = IS净收入）
- D&A与附表匹配
- SBC与IS匹配
- ΔAR、Δ存货、ΔAP与WC附表匹配
- 资本支出与DA附表匹配

**第4节：留存收益**
- RE结转检查：前期RE + NI + SBC - 股息 = 期末RE
- 显示调试的组件 breakdown

**第5节：营运资金**
- AR、存货、AP与BS匹配
- DSO、DIO、DPO合理性检查（如果超出正常范围则标记）

**第6节：债务附表**
- 总债务与BS匹配（流动 + 长期债务）
- 利息计算与IS匹配

**第6b节：权益融资**
- 权益发行收益与BS普通股/APIC增加匹配
- 权益现金增加 = 权益账户增加（必须平衡）
- 权益筹集匹配：Δ普通股/APIC（BS） = 权益发行（CFF）（必须 = 0）
- 第0年权益匹配：权益筹集（第0年） = 期初权益资本（第1年）

**第6c节：NOL附表**
- 期初NOL（第1年/成立） = 0（新业务从零NOL开始）
- NOL仅在EBT < 0时增加（必须实现亏损才能产生NOL）
- DTA与BS匹配（NOL附表DTA = BS递延所得税资产）
- NOL利用率 ≤ EBT的80%（2017年后联邦限制）
- NOL余额非负（不能使用超过可用量）
- NOL仅在EBT < 0时产生
- 当应税收入 ≤ 0时，税收支出 = 0

**第7节：情景层次**
- 绝对指标：上行 > 基准 > 下行（NI、EBITDA、FCF）
- 边际：上行 > 基准 > 下行（GM%、EBITDA%、NI%）
- 信用指标：上行 < 基准 < 下行（杠杆，反向）

**第8节：公式完整性**
- COGS、S&M、G&A、R&D、SBC由收入百分比驱动（无硬编码）
- 预测年份的公式一致
- 无#REF!、#DIV/0!、#VALUE!错误

**第9节：信用指标阈值**
- 根据covenant阈值将指标标记为绿色/黄色/红色
- 任何红旗信号的摘要

### 主检查公式

将所有部分状态汇总到单个主检查中：
- 如果所有部分通过 → "✓ ALL CHECKS PASS"
- 如果任何部分失败 → "✗ ERRORS DETECTED - REVIEW BELOW"

### 快速调试工作流程

当主状态显示错误时：
1. 滚动查找红色突出显示的部分
2. 识别哪个检查类别有失败
3. 导航到源标签进行调查
4. 修复根本问题
5. 返回检查标签验证解决方案
