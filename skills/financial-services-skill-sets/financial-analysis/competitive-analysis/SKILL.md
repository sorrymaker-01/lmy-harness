# 竞争格局映射

---

## 关键标准 - 适用于所有分析

### 源文件优先

当提供源文件（Excel/CSV）时：
* **直接提取值** — 不要进行自己的计算；使用显示的数字
* **保持一致性** — 当同一指标出现在多个地方时，确保整个过程中值相同
* **验证计算** — 如果提示要求计算，请验证结果与源中的相关数据匹配
* **仅按显示舍入** — 使用与源文件相同的小数精度

### 提示保真度

当提示指定确切要求时，严格按照要求执行：

**幻灯片标题和部分名称：**
* 如果提示说 `"Overview and Competitive Scope" slide` — 完全使用该标题，不要改述
* 如果提示说 `within the "Segment Mix" section` — 完全使用 "Segment Mix" 作为部分标题
* 永远不要用创意替代（例如，当指定 "Overview and Competitive Scope" 时，不要使用 "FY2024 Segment Contribution Analysis"）

**图表 vs 表格：**
* 如果提示说 `embedded chart` — 创建实际的PowerPoint图表对象，而不是表格
* 如果提示说 `data labels must display` — 这些应放在图表元素（条形、切片、线条）上，而不是表格单元格上
* 表格和图表不可互换 — 严格使用指定的格式

**完整数据系列：**
* 如果提示列出7个竞争对手，包括所有7个 — 不是5个或6个
* 如果提示显示2015-2025年的数据，包括所有年份 — 不是子集
* 如果提示指定图表中的6个系列（Uber、Lyft、DiDi、Bolt、Grab、Gojek），包括所有6个 — 不是4个

**精确值和措辞：**
* 如果提示说 `Revenue: $43.98B (+18% YoY)` — 完全显示该格式
* 如果提示说 `surpasses DoorDash 4:1, Lyft 8:1` — 使用这些精确比率，而不是 "7.6x Lyft"
* 如果提示给出特定百分比（例如，"Uber 30%, DiDi 35%"），使用这些精确数字

**如有疑问：** 重新阅读提示。如果它明确指定了什么，那不是建议 — 而是要求。

### 参考文件

此技能在 `references/` 文件夹中包含参考文件。使用方法如下：
* **`references/schemas.md`** — 并购交易、情景分析和幻灯片结构的表格模板。在构建财务表格或投资情景时参考。
* **`references/frameworks.md`** — 按行业分类的2x2矩阵轴对。在选择定位可视化维度时参考。

### 源质量层次

当来源冲突时，按以下顺序优先：
1. **年报/半年报** — 经审计，可靠性最高
2. **业绩说明会 / 投资者演示** — 管理层评论，前瞻性指导
3. **卖方研究** — 分析师估计，对私营公司规模有用
4. **行业报告**（麦肯锡、高德纳等）— 市场规模，趋势
5. **新闻文章** — 仅用于最近发展，根据主要来源验证

### 数据可比性
* **时间段必须匹配** — 所有竞争对手指标来自同一财年。标记例外："(FY24)" vs "(H1 2024)"
* **指标定义必须匹配** — 所有竞争对手使用相同的计算方法
* **货币标准化** — 国际比较时将所有数字转换为美元；注明使用的汇率和日期
* **使用 "-" 表示缺失数据** — 永远不要留空单元格；对于私营公司，使用 "N/A" 或带有 "[E]" 标志的估计
* **引用每个数字** — 格式："[公司] [文档] ([日期])"
* **源文件保真度** — 当提供Excel/CSV文件时，使用确切给定的值；不要重新计算或与显示不同的舍入

### 设计和格式
* **幻灯片标题 = 洞察** — "规模领导者与利基玩家拉开距离" 而非 "竞争分析"
* **幻灯片标题必须适合** — 一行或两行可以，但不要溢出；必要时减小字体大小（最小24pt）
* **路标 = 量化** — "利润率低于40%" 而非 "利润率下降"
* **评级包括实际值** — "●●● $160B" 而非仅 "●●●"
* **需要幻灯片编号** — 每张幻灯片必须有页码

### 演示特定要求
* **需要实际嵌入图表** — 饼图、条形图和线图必须是真正的PowerPoint图表对象（通过pptx技能创建），而不是文本/ASCII表示
* **匹配提示结构** — 如果提示指定幻灯片结构，请遵循
* **竞争对手表格** — 对于全面分析：每个竞争对手的指标表格 + 定性表格。对于快速评估：单个组合表格是可接受的。
* **细分财务** — 尽可能包括收入和EBITDA。对于私营竞争对手或有限披露，仅收入表格是可接受的；注明 "[EBITDA not disclosed]"

### 视觉参考

**查看 `examples/Toast_Embedded_Payments_Competitive_Landscape_v3.pptx` 并匹配其专业质量：**

**间距和溢出预防：**
- **标题到内容的间隙** — 幻灯片标题底部和第一个内容元素之间至少0.4"
- **部分标题间隙** — 部分标题和下面内容之间至少0.25"
- **元素缓冲区** — 任何两个元素（表格、文本框、图表）之间至少0.2"
- **边距安全** — 所有内容离幻灯片边缘至少0.5"
- **文本溢出** — 如果文本不适合，减小字体大小或跨幻灯片分割；永远不要让文本裁剪或重叠

**幻灯片标题：**
- **必须适合幻灯片宽度** — 一行或两行可以，但文本不得溢出或裁剪
- **如果标题太长** — 缩短措辞或减小字体大小（最小24pt）
- **前置洞察** — 将关键点放在前面，细节放在后面

**图表格式：**
- **图例在布局内** — 始终设置 `include_in_layout=True` 以便图例不与图表区域重叠
- **图例位置** — 饼图使用右侧（≤6项），线/条形图使用底部（≤4系列）
- **系列太多** — 如果>6系列，考虑拆分为多个图表或使用表格
- **数据标签** — 对于饼图，在切片上显示百分比，而不仅仅依赖图例

**排版（明确设置，永远不要使用默认值）：**
- 幻灯片标题：28-32pt粗体
- 部分标题：16-20pt粗体
- 正文：11-14pt常规
- 表格文本：10-12pt常规
- 来源/脚注：8-10pt，灰色
- **一致性规则**：同一元素类型 = 整个演示文稿中相同的字体大小

**布局：**
- 干净的网格对齐 — 表格和文本块对齐到一致的边距
- 充足的空白 — 不要拥挤幻灯片；让内容呼吸
- 视觉层次 — 最重要的洞察最大/最突出
- 每张幻灯片一个关键信息 — 支持细节在下面

**颜色：**
- 有限的调色板 — 最多2-3种颜色（一种强调色用于强调）
- 柔和的色调 — 避免明亮/饱和的颜色；使用深蓝色、灰色、柔和的蓝色
- 一致的应用 — 整个过程中相同的颜色含义（例如，关键指标的强调色）

**表格：**
- 浅灰色标题行，粗体文本
- 交替行阴影（微妙）或干净的白色，细边框
- 数字右对齐，文本左对齐
- 充分的单元格填充 — 文本不应接触边框

**评级视觉：**
- ●●● / ●●○ / ●○○ 系统，旁边有实际指标
- 在比较表格中一致放置

调整结构和指标以适应您的行业 — 但保持这种专业水平。

### 什么是严格的 vs 灵活的

| 严格（每次） | 灵活（视情况而定） |
|---------------------|------------------------|
| 当提示指定时，使用确切的标题/部分 | 当提示未指定时，使用创意标题 |
| 提示说图表时使用图表；提示说表格时使用表格 | 当提示未指定时，可视化类型 |
| 包括提示中列出的所有数据点/竞争对手 | 当提示未指定时，竞争对手数量 |
| 当提示指定时，使用确切的值/比率 | 当提示未指定精度时，舍入 |
| 标题适合不溢出 | 竞争对手类别数量 |
| 元素之间的最小间距 | 比较哪些维度 |
| 图表图例在布局内 | 分析的竞争对手数量 |
| 无重叠文本/元素 | 可视化类型（2x2、雷达、层级） |

## 工作流阶段

### 阶段1：澄清要求
开始前，确认：
* **范围**：单个公司深度分析还是多公司比较？
* **输出**：演示文稿还是书面备忘录？
* **重点领域**：特定竞争对手、维度或战略问题？
* **投资背景**：需要情景/路标？
* **源文件**：提供了哪些数据文件，应提取哪些值？

### 阶段2：研究 → 大纲 → 审查 → 创建
**在大纲审查之前不要创建最终输出。**

下面的10步分析工作流（步骤0-9）在阶段2执行。在创建最终幻灯片或文档之前，完成研究和大纲。

## 分析工作流

### 步骤0：确定行业定义指标
在深入分析之前，确定对该行业最重要的3-5个指标：

| 行业 | 关键指标 |
|----------|-------------|
| SaaS | ARR、NRR、CAC回报期、LTV/CAC、40规则 |
| 支付 | GPV、收取率、附加率、交易利润率 |
| 市场平台 | GMV、收取率、买家/卖家比率、重复率 |
| 零售 | 同店销售、库存周转率、每平方英尺销售额 |
| 物流 |  volume、单位成本、准时交付率、容量利用率 |

对于未列出的行业，确定投资者和运营商用于基准绩效的3-5个指标。

在所有竞争对手比较中一致使用这些指标。

### 步骤1：市场背景
- 市场规模（当前和预测）及来源
- 增长驱动因素和阻力
- 重塑行业的关键趋势

**正确：** "嵌入式支付市场2024年为80-100B美元，以20-25%的CAGR增长（麦肯锡2024）"
**错误：** "市场很大且增长迅速"

### 步骤2：行业经济学

映射价值流。方法因行业类型而异：
* **垂直结构** — 价值链层，每层典型利润率
* **平台/网络** — 生态系统参与者和它们之间的价值流
* **分散** — 整合动态和按规模的利润率差异

### 步骤3：目标公司简介

```
| 指标 | 值 |
|--------|-------|
| 收入 | $4.96B |
| 增长 | +26% YoY |
| 毛利率 | 45% |
| 盈利能力 | $373M 调整后EBITDA |
| 客户 | 134K |
| 留存率 | 92% |
| 市场份额 | ~15% |
```

**对于多细分公司，添加细分细分：**
```
| 细分 | 收入 | 收入YoY | 收入% | EBITDA | EBITDA YoY | 利润率 |
|---------|---------|---------|-------|--------|------------|--------|
| 细分A   | $25.1B  | +26%    | 57%   | $6.5B  | +31%       | 26%    |
| 细分B   | $13.8B  | +31%    | 31%   | $2.5B  | +64%       | 18%    |
| 细分C   | $5.1B   | -2%     | 12%   | -$74M  | -16%       | -1%    |
| 总计   | $44.0B  | +18%    | 100%  | $6.5B* | -          | 15%    |
```
*如适用，注明企业成本

### 步骤4：竞争对手映射

使用适合的框架对竞争对手进行分组：
* **按商业模式** — 平台 vs 垂直 vs 水平
* **按细分** — 企业 vs 中小企业 vs 消费者  
* **按姿态** — 直接 vs 相邻 vs 新兴
* **按起源** —  incumbent vs 颠覆者 vs 新进入者

### 步骤5：定位可视化

| 可视化 | 最适合 |
|--------------|----------|
| 2x2矩阵 | 两个主导竞争因素 |
| 雷达/蜘蛛图 | 多因素比较 |
| 层级图 | 自然聚类/战略群体 |
| 价值链映射 | 垂直行业 |
| 生态系统映射 | 平台市场 |

### 步骤6：竞争对手深度分析

**表1 — 指标：**
```
| 指标 | 值 |
|--------|-------|
| 收入 | $X.XB |
| 增长 | +XX% YoY |
| 毛利率 | XX% |
| 市值 | $X.XB |
| 盈利能力 | $XXXM EBITDA |
| 客户 | XXK |
| 留存率 | XX% |
| 市场份额 | ~XX% |
```

**表2 — 定性：**
```
| 类别 | 评估 |
|----------|------------|
| 业务 | 他们做什么（1句话） |
| 优势 | 2-3个要点 |
| 劣势 | 2-3个要点 |
| 战略 | 当前优先事项 |
```

### 步骤7：比较分析

```
| 维度 | 公司A | 公司B | 公司C |
|-----------|-----------|-----------|-----------|
| 规模 | ●●● $160B | ●●○ $45B | ●○○ $8B |
| 增长 | ●●○ +26% | ●●● +35% | ●●○ +22% |
| 利润率 | ●●○ 7.5% | ●○○ 3.2% | ●●● 15% |
```

### 步骤8：战略背景
- 并购交易（倍数，战略理由）
- 合作伙伴关系和整合趋势
- 资本筹集模式
- 监管发展

### 步骤9：综合

**竞争护城河评估：**
使用这些类别评估每个竞争对手的持久优势：

| 护城河类型 | 评估内容 |
|-----------|----------------|
| 网络效应 | 用户/供应商飞轮强度；跨边 vs 同边效应 |
| 转换成本 | 技术集成深度，合同锁定，行为习惯 |
| 规模经济 | 规模下的单位成本优势；最小有效规模 |
| 无形资产 | 品牌价值，专有数据，监管许可证，专利 |

将每个评为强/中/弱，并提供支持证据。

**必需的综合元素：**
- 持久优势（难以复制）— 映射到上述护城河类别
- 结构性弱点（难以修复）
- 当前状态 vs 轨迹

**对于投资背景：**
```
| 情景 | 概率 | 关键驱动因素 |
|----------|-------------|------------|
| 乐观 | 30% | 市场份额增长，利润率扩张 |
| 基准 | 50% | 当前轨迹继续 |
| 悲观 | 20% | 竞争压力，利润率压缩 |
```
