# 模型检查器

## 工作流程

### 步骤1：导入模型

- 接受用户的Excel模型（.xlsx或.xlsm）
- 识别模型类型：DCF、LBO、并购、三表、可比公司、回报或自定义
- 映射结构：存在哪些标签，它们如何链接，输入与输出在哪里

### 步骤2：结构检查

**标签和布局审查：**
- 输入是否与计算明确分开？
- 是否有一致的颜色编码约定？（蓝色 = 输入，黑色 = 公式，绿色 = 链接）
- 是否有隐藏的标签或行可能包含覆盖？
- 模型流程是否逻辑？（假设 → 损益表 → 资产负债表 → 现金流量表 → 估值）

**公式一致性：**
- 检查公式内的硬编码数字（部分硬编码）
- 检查行/列范围的公式不一致（应该是相同的公式拖放）
- 识别任何#REF!、#VALUE!、#N/A、#DIV/0!错误
- 标记格式化为公式但包含硬编码值的单元格

### 步骤3：完整性检查

**资产负债表：**
- 总资产 = 总负债 + 权益（每个期间）
- 如果不平衡，量化差距并追踪断点
- 检查留存收益是否正确结转：前期RE + 净收入 - 股息 = 当前RE
- 验证商誉和无形资产是否来自收购假设（如果是并购模型）

**现金流量表：**
- 现金流量表的期末现金 = 资产负债表的现金（每个期间）
- 经营现金流 + 投资现金流 + 融资现金流 = 现金变化
- 现金流量表的D&A与损益表的D&A匹配
- 现金流量表的资本支出与资产负债表的PP&E结转匹配
- 现金流量表的营运资金变化与资产负债表变动匹配（应收账款、应付账款、存货）

**损益表：**
- 收入构建与细分/产品详情匹配
- 销货成本和毛利率与假设一致
- 税收支出 = 税前收入 × 税率（检查递延税调整）
- 股份数量与稀释计划匹配（期权、可转换债券、回购）

**循环引用：**
- 检查循环引用（利息支出 → 债务余额 → 现金 → 利息）
- 如果是有意的（在LBO/三表模型中常见），验证迭代切换是否有效
- 如果是无意的，追踪循环并建议如何打破它

### 步骤4：逻辑检查

**合理性：**
- 增长率是否合理？（100%+收入增长无解释 = 红旗信号）
- 边际是否在行业规范范围内？标记异常值
- 终端价值是否主导DCF？（终端价值占企业价值>75%是黄色信号）
- 预测是否不切实际地呈曲棍球棒状？
- EBITDA增长到第10年是否复合到荒谬的数字？

**敏感性和边缘情况：**
- 0%增长时会发生什么？负增长？
- 模型在负EBITDA时是否崩溃？
- 杠杆比率是否为负或超出实际界限？
- 是否有任何除以零的风险？

**跨标签一致性：**
- 链接的单元格是否实际匹配其来源？（复制粘贴错误很常见）
- 所有标签上的日期标题是否一致？
- 单位是否匹配（千 vs. 百万 vs. 实际值）？

### 步骤5：按模型类型分类的常见错误

**DCF：**
- 贴现率应用于错误期间（年中 vs. 年末惯例）
- 终端价值未正确贴现
- WACC使用账面价值而非市场价值
- 自由现金流包含利息支出（应无杠杆）
- 税收盾重复计算

**LBO：**
- 债务偿还与现金扫掠机制不匹配
- PIK利息未累计到债务余额
- 管理层滚投未反映在回报中
- 退出倍数应用于错误的EBITDA（最近12个月 vs. 未来12个月）
- 费用和开支未从第1天权益中扣除

**并购模型：**
-  accretion/dilution使用错误的股份数量（交易前 vs. 交易后）
- 协同效应未正确分阶段实施
- 购买价格分配不平衡
- 未包括现金的放弃利息
- 交易费用不在资金来源与用途中

**三表：**
- 营运资金变化符号约定错误
- 折旧与PP&E附表不匹配
- 债务到期附表与本金支付不匹配
- 支付的股息超过净收入无解释

### 步骤6：报告

生成模型审计报告：

**摘要：**
- 模型类型和整体评估（干净 / 次要问题 / 主要问题）
- 按严重程度发现的问题数量

**问题日志：**

| # | 标签 | 单元格/范围 | 严重程度 | 类别 | 描述 | 建议修复 |
|---|-----|-----------|----------|----------|-------------|--------------|
| 1 | | | 严重/警告/信息 | 公式/逻辑/平衡/硬编码 | | |

**严重程度定义：**
- **严重**：模型产生错误输出（资产负债表不平衡，公式损坏）
- **警告**：模型工作但存在风险（硬编码，不一致的公式，边缘情况失败）
- **信息**：风格和最佳实践建议（颜色编码，布局，命名）

### 步骤7：输出

- 问题日志表（在聊天或Excel中）
- 带有标记单元格注释的带注释模型（如果用户提供文件）
- 带修复优先级的摘要评估

## 重要注意事项

- 始终首先检查资产负债表平衡 — 如果不平衡，在平衡之前其他都不重要
- 硬编码覆盖是模型错误的#1来源 — 积极搜索它们
- 符号约定错误（现金流出的正数 vs. 负数）非常常见
- "工作"的模型仍然可能错误 — 对照行业基准检查输出
- 如果模型使用VBA宏，注意任何无法仅从公式审计的宏驱动计算
- 不要未经询问就更改模型 — 报告问题并让用户决定如何修复
