# 投资银行演示文稿检查器

从四个维度对投资银行演示文稿进行全面质量控制。

## 前提条件

在检查前提取演示文稿内容：
```bash
python -m markitdown presentation.pptx > content.md
```

对于视觉检查，使用 `pptx` 技能工作流程转换为图像。

## 检查工作流程

### 1. 数字一致性

提取带有幻灯片引用的数字：
```bash
python scripts/extract_numbers.py content.md --check
```

验证：
- 关键指标在所有幻灯片中一致（收入、EBITDA、倍数）
- 计算正确（总计、百分比、增长率）
- 单位一致（使用相同规模：百万 vs 十亿，% vs 基点）
- 单位格式一致（例如，$M vs $MM，$B vs $Bn - 全程使用一种风格）
- 时间段对齐（财年 vs 最近12个月 vs 季度）

标记模式：
```
问题：收入不匹配
- 幻灯片3、8：$500M
- 幻灯片15（DCF输入）：$485M
行动：核对数字
```

### 2. 数据与叙述一致性

将声明映射到支持数据：
- 趋势陈述 → 图表方向
- 市场地位声明 → 收入/份额数据
- 事实断言 → 验证准确性

标记矛盾：
```
问题：叙述与数据矛盾
- 幻灯片4："利润率下降"
- 幻灯片7图表：利润率从18% → 22%
行动：更新叙述或验证数据
```

检查合理性（例如，"1000亿美元市场中的#1玩家"但收入仅2亿美元 = 0.2%份额）。

### 3. 语言润色

扫描：
- 随意措辞（"相当好"、"很多"）
- 没有具体数字的模糊量词
- 缩写、感叹号
- 不一致的术语

请参阅[references/ib-terminology.md](references/ib-terminology.md)获取替换模式。

标记模式：
```
问题：随意语言（幻灯片12）
- "这笔交易是显而易见的"
→ "该交易呈现出引人注目的价值主张"
```

### 4. 格式质量控制

审核每张幻灯片：
- **图表**：来源引用、轴标签、图例
- **排版**：一致的字体、大小层次
- **数字**：一致的格式（1,000 vs 1K）
- **日期**：全程一致的格式
- **脚注**：适当的来源和免责声明

## 输出

使用[references/report-format.md](references/report-format.md)中的模板呈现结果。

按严重程度分类：
- **关键**：数字不匹配、事实错误
- **重要**：语言、叙述一致性
- **次要**：格式不一致
