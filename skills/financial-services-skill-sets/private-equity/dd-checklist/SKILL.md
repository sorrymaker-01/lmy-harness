# 尽职调查清单
## 工作流程

### 步骤1：确定尽职调查范围

向用户询问：
- **目标公司**：名称、行业、商业模式
- **交易类型**：平台收购、附加收购、成长股权、资本重组、企业剥离
- **交易规模/复杂性**：决定尽职调查的深度
- **关键关注点**：任何需要优先考虑的已知问题（客户集中度、监管、环境等）
- **时间线**：意向书/交割的目标时间是什么时候？

### 步骤2：生成工作流程清单

生成针对行业的所有主要工作流程清单：

**财务尽职调查**
- 盈利质量（QoE）— 收入和EBITDA调整
- 营运资金分析 — 标准化与实际
- 债务和类债务项目
- 资本支出（维护 vs 增长）
- 税务结构和风险敞口
- 审计历史和会计政策
- 预计调整（运行率、协同效应）

**商业尽职调查**
- 市场规模和增长（TAM/SAM/SOM）
- 竞争定位和市场份额
- 客户分析 — 集中度、留存率、NPS
- 定价能力和合同结构
- 销售渠道和积压订单
- 市场进入有效性

**法律尽职调查**
- 公司结构和组织结构图
- 重要合同（客户、供应商、合作伙伴）
- 诉讼历史和未决索赔
- 知识产权组合和保护
- 监管合规
- 就业协议和非竞争条款

**运营尽职调查**
- 管理团队评估
- 组织结构和关键人员风险
- IT系统和基础设施
- 供应链和供应商依赖
- 设施和房地产
- 保险覆盖

**人力资源/人员尽职调查**
- 组织结构图和人员趋势
- 薪酬基准
- 福利和养老金义务
- 关键员工留存风险
- 文化评估
- 工会/劳工协议

**IT/技术尽职调查**（适用于技术驱动型企业）
- 技术栈和架构
- 技术债务评估
- 网络安全状况
- 数据隐私合规（GDPR、CCPA、SOC2）
- 产品路线图和研发支出
- 可扩展性评估

**环境/ESG**（如适用）
- 环境负债
- 监管合规历史
- ESG风险和机会

### 步骤3：状态跟踪

对每个项目进行跟踪：

| 项目 | 工作流程 | 优先级 | 状态 | 负责人 | 备注 |
|------|-----------|----------|--------|-------|-------|
| QoE报告 | 财务 | P0 | 待处理 | | |
| 客户访谈 | 商业 | P0 | 进行中 | | 10个完成3个 |

状态选项：未开始 → 请求中 → 已收到 → 审查中 → 完成 → 红旗信号

### 步骤4：红旗信号摘要

维护在尽职调查过程中发现的红旗信号列表：
- 发现了什么
- 哪个工作流程
- 严重程度（交易终止 / 重大 / 可管理）
- 缓解措施或解决路径
- 对估值或交易条款的影响

### 步骤5：输出

- 每个工作流程有标签的Excel工作簿（默认）
- 摘要仪表板：按工作流程的完成百分比、未完成项目、红旗信号
- 交易团队的每周状态更新格式

## 行业特定补充

根据行业自动添加相关项目：
- **软件/SaaS**：ARR质量、队列分析、托管成本、SOC2
- **医疗保健**：监管批准、报销风险、付款人组合
- **工业**：设备状况、环境修复、安全记录
- **金融服务**：监管资本、合规历史、信贷质量
- **消费品**：品牌健康、渠道组合、季节性、库存管理

## 重要注意事项

- 优先处理阻碍意向书或交割的P0项目
- 标记卖方响应缓慢的项目 — 可能表明存在问题
- 将数据室内容与清单交叉引用以识别差距
- 随着尽职调查的进展更新清单 — 这是一个动态文档
