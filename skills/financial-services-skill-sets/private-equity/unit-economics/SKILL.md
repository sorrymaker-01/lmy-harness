# 单位经济学分析

## 工作流程

### 步骤1：识别业务模型

确定收入模型以定制分析：
- **SaaS/订阅**：ARR、净留存率、cohort
- **经常性服务**：合同价值、续约率、向上销售
- **交易/使用量基础**：每笔交易收入、交易量趋势、抽成率
- **混合**：按收入流分解

### 步骤2：核心指标

#### ARR/收入质量
- **ARR桥接**：期初ARR → 新增 → 扩张 → 收缩 → 流失 → 期末ARR
- **按cohort的ARR**：年份分析 — 每个年度cohort如何留存和增长？
- **收入集中度**：前10/20/50客户占总额的百分比
- **按类型的收入**：经常性 vs. 非经常性 vs. 专业服务
- **合同结构**：ACV分布、多年期百分比、自动续约百分比

#### 客户经济学
- **CAC（客户获取成本）**：总销售和营销支出 / 新获取客户数
- **LTV（生命周期价值）**：(ARPU × 毛利率) / 流失率
- **LTV:CAC比率**：健康业务目标 >3x
- **CAC回收期**：收回获取成本的月数
- **混合vs. 细分**：按客户细分（企业vs. SMB vs. 中端市场）分解

#### 留存与扩张
- **毛留存率**：期初ARR留存的百分比（不包括扩张）
- **净留存率（NDR）**：期初ARR留存的百分比（包括扩张）
- **Logo流失**：流失客户的百分比
- **收入流失**：流失收入的百分比（通常与logo流失不同）
- **扩张率**：向上销售+交叉销售占期初ARR的百分比

#### Cohort分析
构建cohort矩阵，显示：

| 2020 | $1.0M | $1.1M | $1.2M | $1.1M | |
| 2021 | $1.5M | $1.7M | $1.8M | | |
| 2022 | $2.0M | $2.3M | | | |
| 2023 | $3.0M | | | | |

同时显示绝对值和指数化（第0年=100%）视图。

#### 利润率瀑布
- 收入 → 毛利润 → 贡献毛利 → EBITDA
- 完全加载的单位经济学：获取、服务和保留客户的成本是多少？
- 按收入流的毛利率（订阅 vs. 服务 vs. 其他）

### 步骤3：基准比较

将单位经济学与相关基准进行比较：
- **SaaS 40规则**：增长率 + EBITDA利润率 > 40%
- **SaaS 魔术数字**：净新增ARR / 前期销售和营销支出 > 0.75x
- **NDR基准**：最佳 >120%，良好 >110%，令人担忧 <100%
- **LTV:CAC**：最佳 >5x，良好 >3x，令人担忧 <2x
- **毛留存率**：最佳 >95%，良好 >90%，令人担忧 <85%
- **CAC回收期**：最佳 <12个月，良好 <18个月，令人担忧 >24个月

## 重要注意事项

- 如果有可用的原始客户级数据，始终要求提供 — 聚合指标可能隐藏问题
- 如果扩张足够强，NDR高于100%可以掩盖高毛流失率 — 始终同时显示两者
- Cohort分析是收入质量最重要的视图 — 努力获取这些数据
- 区分合同ARR和实际确认收入
- 对于使用量基础模型，关注消费趋势和扩张模式，而非传统ARR指标
- 专业服务收入应单独评估 — 它不是经常性的，利润率通常较低
