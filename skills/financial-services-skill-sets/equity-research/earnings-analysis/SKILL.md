# 股票研究盈利更新

创建专业的**盈利更新报告**，分析已覆盖公司的季度业绩，遵循国内机构标准（中金、华泰、海通等格式），适用于A股、港股及中概股上市公司。

**关键特征：**
- **长度**：8-12页
- **字数**：3,000-5,000字
- **表格**：1-3个摘要表格（非全面）
- **图表**：8-12个图表
- **周转时间**：1-2天（盈利发布后24-48小时内）
- **受众**：已经熟悉公司的客户
- **重点**：新内容 - 业绩超出/低于预期、更新后的估计、对投资论点的影响
- **字体**：全程使用Times New Roman（除非用户另有指定）

## 使用时机

当用户请求以下内容时使用：
- "为[公司] 2024年第三季度创建盈利更新"
- "分析[公司]的季度业绩"
- "[公司]的盈利后报告"
- "[公司]的Q1/Q2/Q3/Q4更新"

**不适用情况：**
- 用户请求"首次覆盖报告" → 使用不同技能
- 用户请求"闪速报告"或"快速解读" → 不同格式
- 公司尚未被覆盖 → 需要先进行首次覆盖

## 关键要求

### 1. 速度与及时性
- 在盈利发布后24-48小时内发布
- 仅关注新信息
- 不要过多重复公司背景

### 2. 业绩超出/低于预期分析
- 首先说明公司业绩是超出还是低于预期
- 量化差异（例如，"收入超出1.2亿元人民币或3%"）
- 解释业绩与预期不同的原因

### 3. 摘要格式
- 表格限制为1-3个（仅摘要，非全面）
- 不包含完整的损益表/现金流量表/资产负债表（仅关键指标）
- 假设读者已看过首次覆盖报告

### 4. 引用与来源归因 ⭐⭐⭐ 必须

**重要**：使用具体来源和可点击超链接正确引用所有数据。

**在每个图表和表格中包含带有可点击链接的具体引用：**

```
来源：2024年第三季度10-Q文件，2024年11月8日提交；公司盈利发布
        [将"10-Q"超链接至：https://www.sec.gov/cgi-bin/viewer?accession=...]
        [将"盈利发布"超链接至：https://investor.company.com/news/q3-2024]
```

**Word中链接应如何显示：**
- 文档名称显示为蓝色、带下划线的可点击链接
- 读者可按住Ctrl+点击直接打开来源
- 不是纯文本URL - 带有显示文本的格式化超链接

**必需的来源列表：**

在每个盈利更新中引用：
- ✅ 盈利发布（带日期和URL）
- ✅ 10-Q文件（带提交日期和EDGAR链接）
- ✅ 盈利电话会议记录（带日期）
- ✅ 投资者演示/补充材料（如有）
- ✅ 共识估计来源（彭博/ FactSet等，带日期）
- ✅ 先前的指引（来自上一季度的材料）

**带有可点击超链接的参考部分：**

在报告末尾包含"来源"部分：

```
来源与参考

盈利材料（2024年第三季度）：
• 盈利发布（2024年11月7日）
  [将整行超链接至：https://investor.company.com/news/q3-2024-earnings]

• 10-Q表格（2024年11月8日提交）
  [超链接至：https://www.sec.gov/cgi-bin/viewer?accession=...]

• 盈利电话会议记录（2024年11月7日）
  [超链接至：https://seekingalpha.com/article/...]

• 投资者演示（2024年11月7日）
  [超链接至：https://investor.company.com/presentations/q3-2024.pdf]
```

**验证检查清单：**
- [ ] 每个图表都有带具体文档和日期的来源
- [ ] 每个表格都有带文档引用的来源
- [ ] 业绩超出/低于预期分析引用带日期的共识来源
- [ ] 指引变更引用当前和先前的指引来源
- [ ] 关键统计数据有脚注
- [ ] 来源部分列出所有带URL的材料
- [ ] 所有URL都是可点击超链接（非纯文本）
- [ ] 所有SEC文件都超链接至EDGAR查看器

### 5. 更新后的估计
- 根据业绩更新前瞻性估计
- 清晰显示旧估计与新估计
- 解释变更内容及原因

## 高级工作流程

盈利更新流程遵循5个阶段：

### 阶段1：数据收集（30-60分钟）

**🚨🚨🚨 重要：训练数据已过时 🚨🚨🚨**

**开始前 - 按顺序完成以下4个步骤：**
1. **检查今天的日期** - 写下当前日期
2. **搜索最新信息** - 使用网络搜索："[公司]最新盈利结果"
3. **验证日期** - 确认盈利发布在过去3个月内
4. **检查记录日期** - 验证电话会议记录日期与发布日期匹配

**常见错误**：使用训练数据中的过时盈利电话会议，而不是搜索最新信息。

**要求：**
- ✅ 搜索最新盈利信息 - 不要依赖训练数据
- ✅ 写下今天的日期和找到的发布日期
- ✅ 验证发布日期在今天的3个月内
- ✅ 验证电话会议记录日期与发布日期匹配
- ✅ 如果日期不匹配或已过时（>3个月），重新搜索

**详见 [references/workflow.md](references/workflow.md)** 了解详细的搜索程序和验证步骤。

### 阶段2：分析（2-3小时）
- 每个关键指标的超出/低于预期分析
- 细分/地理/产品分类
- 利润率和指引分析
- 更新财务模型和估计

**详见 [references/workflow.md](references/workflow.md)** 了解详细的分析框架。

### 阶段3：图表生成（1-2小时）
创建8-12个关注季度趋势和新内容的图表：
- 季度收入进展
- 季度每股收益进展
- 季度利润率趋势
- 按细分/地理划分的收入
- 关键运营指标
- 超出/低于预期摘要
- 估计修订
- 估值图表

**详见 [references/workflow.md](references/workflow.md)** 了解图表规格。

### 阶段4：报告创建（2-3小时）
创建8-12页的DOCX报告，具有特定结构。

**详见 [references/report-structure.md](references/report-structure.md)** 了解完整的逐页模板和格式要求。

**高级结构：**
- 第1页：带评级和目标价格的盈利摘要
- 第2-3页：详细业绩分析
- 第4-5页：关键指标和指引
- 第6-7页：更新的投资论点
- 第8-10页：估值和估计
- 第11-12页：附录（可选）

### 阶段5：质量检查与交付（30分钟）
在交付前验证内容、格式、准确性和及时性。

**详见 [references/best-practices.md](references/best-practices.md)** 了解质量检查清单和需避免的常见错误。

## 输出规格

**主要交付物**：DOCX报告（8-12页）
**文件名**：`[公司]_Q[季度]_[年份]_Earnings_Update.docx`
**示例**：`Nike_Q2_FY24_Earnings_Update.docx`

**内容：**
- 第1页：带评级、目标价格和关键要点的摘要
- 第2-3页：详细业绩分析
- 第4-5页：关键指标和指引
- 第6-7页：更新的论点评估
- 第8-10页：估值和估计
- 第11-12页：附录（可选）
- 8-12个嵌入图表
- 1-3个摘要表格
- 带可点击超链接的完整来源部分

**可选交付物**：XLS模型更新（盈利更新的可选内容）
