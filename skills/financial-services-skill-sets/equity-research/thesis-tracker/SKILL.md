# 投资论点追踪

## 工作流程

### 步骤1：定义或加载论点

如果创建新论点：
- **公司**：名称和股票代码
- **持仓**：做多还是做空
- **论点陈述**：1-2句话核心论点（如"做多ACME——随着组合向软件转移，定价权+经营杠杆带来利润率扩张"）
- **关键支柱**：3-5个支撑论据
- **关键风险**：3-5个会使论点失效的风险
- **催化剂**：可能证明/证伪论点的即将发生的事件（业绩、产品发布、监管决定）
- **目标价/估值**：如果论点实现，价值几何
- **止损触发**：什么会让你退出

如果更新现有论点，向用户询问新数据点或发展情况。

### 步骤2：更新日志

对于每个新数据点或发展：

- **日期**：何时发生
- **数据点**：什么变化了（业绩超预期、管理层离职、竞争对手动向等）
- **论点影响**：这是加强、削弱还是对特定支柱无影响？
- **行动**：不变/加仓/减仓/退出
- **更新后信心**：高/中/低

### 步骤3：论点记分卡

维护持续更新的记分卡：

| 支柱 | 原始预期 | 当前状态 | 趋势 |
|------|---------|---------|------|
| 收入增长>20% | 按计划 | Q3为22% | 稳定 |
| 利润率扩张 | 落后 | 利润率同比持平 | 令人担忧 |
| 新产品发布 | 待定 | 延期至Q2 | 关注 |

### 步骤4：催化剂日历

追踪即将到来的催化剂：

| 日期 | 事件 | 预期影响 | 备注 |
|------|------|---------|------|
| | | | |

### 步骤5：输出

论点总结适用于：
- 早会讨论
- 投资组合回顾
- 风险委员会汇报

格式：简明的markdown或Word文档，包含记分卡、近期更新和当前信心水平。

## 重要说明

- 论点应该是可证伪的——如果没有任何东西能证伪它，那就不是论点
- 像追踪证实证据一样严格追踪证伪证据
- 至少每季度回顾论点，即使没有发生重大事件
- 如果用户管理多个持仓，提议进行完整的投资组合论点回顾
- 以结构化格式存储论点数据，以便跨会话引用
